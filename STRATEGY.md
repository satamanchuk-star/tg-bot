# Стратегия развития Жабота: ИИ-продвижение без потери души

## Философия

Жабот — не безликий AI-ассистент, а **сосед с характером**. Любое улучшение должно проходить тест:
> "Сказал бы это остроумный сосед, который всё знает про дом и любит пошутить?"

Если нет — фича не нужна или её нужно переосмыслить.

---

## Фаза 1: Умная память (2-4 недели)

### 1.1 Персональный контекст жителей

**Что**: Бот запоминает факты о жителях из разговоров (корпус, этаж, имя питомца, интересы).

**Как**:
- Новая модель `ResidentProfile` с полями: `user_id`, `facts` (JSON), `last_updated`
- После каждого диалога AI извлекает факты через lightweight-промпт:
  ```
  "Извлеки факты о пользователе из диалога. JSON: {name, building, floor, pets, interests, family}"
  ```
- Факты используются в system prompt при следующих обращениях
- Пользователь может сказать "забудь обо мне" → очистка профиля (GDPR-совместимо)

**Эффект**: "О, привет! Как твой корги? Кстати, в 3-м корпусе завтра отключат воду — тебя касается."

**Файлы для изменения**:
- `app/models.py` — новая модель `ResidentProfile`
- `app/services/ai_module.py` — промпт извлечения фактов + инъекция в контекст
- `app/handlers/help.py` — команда `/forget_me`

### 1.2 Эмоциональная память чата

**Что**: Бот отслеживает "настроение" чата и адаптирует тон.

**Как**:
- Sentiment уже собирается в `message_logs` (поле `sentiment`)
- Добавить скользящее окно: средний sentiment за последние 50 сообщений
- Если чат злой/напряжённый → бот мягче, больше юмора, мирит
- Если чат весёлый → бот шутит активнее, предлагает игру
- Новый сервис `app/services/mood.py` с методом `get_chat_mood() -> Mood`

**Эффект**: Бот чувствует атмосферу и реагирует уместно, а не механически.

### 1.3 Улучшенная сжатая память (conversation compaction)

**Что**: Вместо простого суммаризирования — извлечение структурированных фактов.

**Как**:
- При компрессии `chat_history` выделять:
  - `resolved_issues` — решённые вопросы (не повторять)
  - `pending_issues` — нерешённые (следить)
  - `user_preferences` — как пользователь любит получать ответы
- Хранить в `chat_history` как отдельный тип записи `type=structured_summary`

**Файлы**: `app/services/chat_history.py`

## Фаза 2: Обучение через диалог (4-8 недель)

### 2.1 Система обратной связи 2.0

**Что сейчас**: Кнопки 👍/👎 на AI-ответы, счётчик в `frequent_questions`.

**Что добавить**:
- **Автокоррекция**: Если пользователь поправляет бота ("нет, аптека на Ленина, а не на Мира"), бот:
  1. Извлекает коррекцию через AI
  2. Обновляет RAG-запись или создаёт новую
  3. Подтверждает: "Записал, спасибо! В следующий раз не ошибусь."
- **Вес источников**: Ответы с высоким рейтингом (+3 и выше) становятся "проверенными"
- **Обучение модерации**: Результаты голосования в `moderation_training` используются для fine-tuning промптов

**Как реализовать**:
```python
# app/services/learning.py — новый сервис

class LearningService:
    async def detect_correction(self, user_msg: str, bot_msg: str) -> Optional[Correction]:
        """AI определяет, поправляет ли пользователь бота."""

    async def apply_correction(self, correction: Correction) -> None:
        """Обновляет RAG/FAQ на основе коррекции."""

    async def update_moderation_rules(self) -> None:
        """Анализирует moderation_training и корректирует пороги."""
```

**Файлы**:
- Новый: `app/services/learning.py`
- Изменить: `app/handlers/help.py` — обработка reply как потенциальной коррекции
- Изменить: `app/services/rag.py` — метод `update_from_correction()`

### 2.2 Краудсорсинг знаний

**Что**: Жители обучают бота естественным образом.

**Механика**:
- Бот замечает, что не знает ответ → спрашивает: "Я пока не знаю. Кто из соседей подскажет?"
- Если кто-то отвечает reply на вопрос → бот предлагает: "Сохранить этот ответ? 👍 — да"
- 3+ голосов 👍 → ответ попадает в RAG как community-verified
- Новая категория RAG: `source=community` (vs `source=admin`)

**Модель доверия**:
- Админы: ответы сохраняются сразу
- Активные жители (100+ сообщений): нужно 2 подтверждения
- Остальные: нужно 3 подтверждения
- Пометка `verified=False` до подтверждения

**Файлы**:
- `app/services/rag.py` — расширить модель RAG полями `source`, `votes`, `verified`
- `app/models.py` — обновить `RagMessage`
- `app/handlers/help.py` — flow сбора ответов от соседей

### 2.3 Адаптивная модерация

**Что**: Модерация учится на решениях сообщества.

**Как**:
- `moderation_training` уже собирает голоса (👍/👎 от админов)
- Добавить: периодический анализ (ежедневный job) накопленных решений
- AI анализирует паттерны ложных срабатываний → корректирует severity-маппинг
- Результат: динамический `severity_overrides` dict в конфиге, обновляемый автоматически
- Логирование всех корректировок для аудита

**Файлы**:
- `app/services/ai_module.py` — метод `recalibrate_moderation()`
- `app/main.py` — ежедневный scheduled job
- `app/models.py` — `ModerationCalibration` для хранения истории корректировок

## Фаза 3: Проактивный сосед (6-12 недель)

### 3.1 Контекстные подсказки

**Что**: Бот сам вступает в разговор, когда может помочь (но не навязчиво).

**Триггеры**:
- Кто-то спрашивает "а где тут...?" → бот подсказывает из Places/RAG
- Новый житель пишет первое сообщение → приветствие + краткий гид
- Тема долго без ответа (>2 часа) → бот пробует ответить
- Повторяющийся вопрос в чате → "Кстати, это уже спрашивали — вот ответ"

**Ограничения (анти-спам)**:
- Максимум 1 проактивное сообщение в час на топик
- Не вмешиваться в активные дискуссии (>5 сообщений за 10 минут)
- Не отвечать на риторические вопросы (AI-классификация)
- Конфиг `PROACTIVE_MODE = True/False` для быстрого отключения

**Реализация**:
```python
# app/services/proactive.py

class ProactiveService:
    async def should_intervene(self, message: Message, topic_id: int) -> bool:
        """Решает, стоит ли боту вмешаться."""

    async def generate_hint(self, message: Message) -> Optional[str]:
        """Генерирует подсказку, если есть релевантная информация."""

    async def greet_newcomer(self, user: User) -> str:
        """Персональное приветствие нового жителя."""
```

**Файлы**:
- Новый: `app/services/proactive.py`
- `app/handlers/moderation.py` — хук на каждое сообщение для proactive check
- `app/config.py` — настройки проактивности

### 3.2 Умные дайджесты

**Что**: Персонализированные дайджесты вместо общих summary.

**Как**:
- На основе `ResidentProfile` (корпус, интересы) фильтровать релевантные события
- Формат: "За сегодня в вашем корпусе: отключение воды завтра, сосед ищет кота, турнир в блэкджек в 22:00"
- Отправка по запросу (`/digest`) или подписка на ежедневные (opt-in)
- AI-суммаризация по топикам, которые пользователю интересны

**Файлы**:
- `app/services/daily_summary.py` — персонализация
- `app/handlers/help.py` — команда `/digest`
- `app/models.py` — `DigestPreference` (подписки)

### 3.3 Мультимодальность

**Что**: Бот понимает фото и голосовые.

**Этапы**:
1. **Фото**: Распознавание скриншотов (показания счётчиков, объявления на подъезде)
   - Через vision-модель (GPT-4o / Claude через OpenRouter)
   - Если житель присылает фото объявления → OCR + сохранение в RAG
2. **Голосовые**: Транскрипция через Whisper API → обработка как текст
   - Пользователь отправляет голосовое → бот отвечает текстом

**Файлы**:
- Новый: `app/services/vision.py`
- Новый: `app/services/speech.py`
- `app/handlers/help.py` — обработка content_type photo/voice

## Фаза 4: Социальный интеллект (3-6 месяцев)

### 4.1 Граф сообщества

**Что**: Бот понимает социальные связи и экспертизу жителей.

**Как**:
- Трекинг: кто кому отвечает, кто эксперт в какой теме
- Модель `CommunityGraph`: `user_id`, `expertise_topics`, `helpfulness_score`
- Когда бот не знает ответ → тегает эксперта: "@ Маша, ты разбиралась в этом — подскажешь?"
- **Приватность**: Только публичные взаимодействия, opt-out через `/dont_tag_me`

### 4.2 Контекст события

**Что**: Бот знает о текущих событиях дома.

**Источники**:
- Сообщения УК (если пересылают в чат) → извлечение: дата, тип (отключение/ремонт/собрание), затронутые корпуса
- Хранение в `HouseEvent` модели с `event_type`, `affected_buildings`, `start_date`, `end_date`
- Автоответы: "Когда дадут воду?" → "По объявлению от 15.03, воду дадут завтра к 18:00 в корпусах 1-3"

### 4.3 Gamification 2.0

**Что**: Расширение игровой системы для вовлечения.

**Идеи**:
- **Достижения**: "Первый ответ", "Помог 10 соседям", "Знаток дома" → значки
- **Сезонные ивенты**: Тематические квизы (Новый год, 1 сентября, день дома)
- **Экономика монет**: Тратить монеты на кастомизацию (титулы в чате через бота)
- **Кооперативные челленджи**: "Корпус 1 vs Корпус 2: кто ответит на больше вопросов квиза"

## Технические принципы

### Выбор модели

| Задача | Текущая модель | Рекомендация |
|--------|---------------|-------------|
| Ассистент | Qwen 3.5 Flash | Оставить — хорошее соотношение цена/качество |
| Модерация | Qwen 3.5 Flash | Оставить — быстрая, дешёвая |
| Извлечение фактов | — | Qwen 3.5 Flash (тот же, lightweight промпт) |
| Vision (фото) | — | Qwen-VL или Claude Haiku (через OpenRouter) |
| Сложные ответы | — | Claude Sonnet (fallback для трудных вопросов) |

**Принцип каскада моделей**: Дешёвая модель → если confidence < 0.7 → дорогая модель. Экономит бюджет.

### Безопасность и приватность

1. **Персональные данные**: Хранить только то, что пользователь сказал публично
2. **Право на забвение**: `/forget_me` удаляет весь профиль
3. **Прозрачность**: `/what_you_know` показывает, что бот знает о пользователе
4. **Модерация AI-ответов**: Все ответы проходят через safety-фильтр (запрет мед/юр/фин советов)
5. **Аудит обучения**: Логировать все корректировки RAG и модерации
6. **Rate limiting**: Сохранить текущие лимиты, не давать AI спамить

### Архитектурные решения

1. **Не fine-tuning, а RAG + промпты**: Fine-tuning дорог и хрупок. RAG + хорошие промпты дают 90% результата
2. **Graceful degradation**: Каждая AI-фича имеет fallback (как сейчас StubAiProvider)
3. **Feature flags**: Каждая новая фича за флагом в конфиге
4. **Метрики**: Трекать для каждой фичи — использование, accuracy, user satisfaction
5. **A/B тестирование**: Для проактивных фич — сначала в одном топике, потом везде

## Чего НЕ делать

1. **Не превращать в ChatGPT-обёртку** — бот должен оставаться Жаботом, а не generic AI
2. **Не отвечать на всё подряд** — молчание лучше бесполезного ответа
3. **Не собирать данные втихую** — прозрачность важнее функциональности
4. **Не усложнять UX** — никаких /команд_со_сложными_параметрами
5. **Не жертвовать скоростью** — ответ за 2 секунды лучше идеального за 10
6. **Не забывать про юмор** — даже сообщения об ошибках должны быть в духе Жабота
