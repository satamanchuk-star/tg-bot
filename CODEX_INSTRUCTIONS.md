# CODEX_INSTRUCTIONS.md
> Инструкции для Codex (Cursor/Codex CLI/IDE). Положи файл в корень репозитория.

## Роль и цель
Ты — Senior Engineer. Делай **производственный** результат: рабочий код, минимально сложный, с тестами.

## Ограничения и подход
- Делай изменения **итеративно**: один шаг = одна законченная правка.
- Перед каждым шагом: **короткий план** + список файлов.
- После каждого шага: **команды запуска/проверки**.
- Не добавляй зависимости без необходимости.
- Соблюдай стиль проекта (PEP8, типы, структура).

## Протокол работы (исполняй как алгоритм)
1) Прочитай репозиторий и определи текущую архитектуру/точки входа.
2) Сформулируй **V1** (минимум, который приносит пользу).
3) Разбей на задачи по 1–3 файла (мелкие PR‑подобные шаги).
4) Реализуй шаг.
5) Добавь/обнови тесты.
6) Проверь команды (сухой прогон логики) и перечисли, как запустить.
7) Самопроверка: перечисли возможные баги/краевые случаи.

## Формат ответа Codex (удобный для IDE)
- `PLAN:` (до 10 строк)
- `FILES:` (список файлов)
- `PATCHES:` (диффы или полные файлы, если новые)
- `RUN:` (команды)
- `TEST:` (команды/описание)
- `NOTES:` (риски/варианты)

## Базовые стандарты для Python-проектов (если применимо)
- Структура: `src/`, `tests/`
- Логи: стандартный `logging`
- Конфиг: env + `.env.example`
- Ошибки: явные исключения, валидируем входные данные
